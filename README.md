# SQL Security Analysis - Applying Filters to Queries

This project demonstrates how SQL queries can be used to investigate potential security issues by analyzing login activity and employee data.

## Objective

To use SQL filtering techniques to retrieve and analyze relevant data from organizational databases in order to identify suspicious activity and support security investigations.

## Scenario

As a cybersecurity professional, I analyzed data from the `employees` and `log_in_attempts` tables to investigate potential security issues involving login activity and employee access.

## Key Analysis Performed

### After-Hours Failed Login Attempts
- Identified failed login attempts outside business hours
- Helped detect potential unauthorized access attempts

### Login Attempts on Specific Dates
- Filtered login activity for targeted investigation
- Focused on identifying unusual behavior on specific days

### Login Attempts Outside of Mexico
- Identified access attempts from non-trusted geographic locations
- Highlighted potential external threats

### Employees by Department
- Retrieved employees in Marketing
- Retrieved employees in Finance or Sales
- Retrieved employees not in IT
